# コーディング時のルール

- 技術スタック、使用ライブラリはREADME.mdを参照する。
- UIデザインはZeplin(https://zpl.io/VK9XynR)を基準に行う。
- 実装はTest Firstで行う。

## Test First実装方法

1. まずVitestの`it.todo`を使って、空のテストケースのみを一通り実装する
2. その状態で開発者にテストケース自体のレビューを依頼する
3. OKが出たら、テストコード自体を実装する
4. テストコードを実行し、失敗することを確認する
5. テストコードに通るように実装を進める

## コーディングにおける禁止事項

- 型定義におけるanyの使用
- useEffectの使用
